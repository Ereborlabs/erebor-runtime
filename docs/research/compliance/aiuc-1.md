# AIUC-1

## What It Is For

Complyance describes AIUC-1 as a standard built for AI-agent security. Treat
AIUC-1 as a real AI-agent control catalog, while still requiring the customer,
auditor, or buyer to provide the exact version and control text they expect.

The useful interpretation for Erebor is "AI use-case control": inventory the
agent use case, define what it may do, enforce boundaries, monitor actual
behavior, and review exceptions.

## Evidence Companies Need

- AI use-case intake and owner
- allowed systems, data, tools, commands, browsers, and SaaS apps
- risk tier and approval path
- runtime monitoring
- approval, denial, exception, and incident records
- periodic recertification of the use case

## What Erebor Can Do

Erebor can make AIUC-1-style controls real at runtime. Instead of trusting a
prompt, it can mediate commands, browser actions, SaaS transitions, API calls,
MCP/tool calls, desktop/network actions, internal-system actions, and high-risk
access when those actions are routed through Erebor-controlled execution paths.
The valuable report is a per-use-case evidence packet showing what the agent
was allowed to do and what it actually did.

## Artifacts Erebor Could Generate

- AI use-case runtime inventory
- approved actions and denied actions report
- human approval and exception log
- agent incident replay packet
- policy conformance report by use case

## Do Not Claim

Do not claim AIUC-1 compliance, certification, or assessor acceptance without
the authoritative control catalog, version, and assessment process. Erebor
supplies runtime evidence and enforcement, not a full AI governance program.
