# ISO/IEC 27018

## What It Is For

ISO/IEC 27018:2025 is guidance for protecting PII in public cloud services when
the cloud service provider acts as a PII processor.

## Evidence Companies Need

- cloud processor responsibility matrix
- customer instruction records
- access-control evidence
- admin activity logs
- subprocessor and SaaS disclosure records
- data location and transfer evidence
- deletion/return workflows
- incident notification evidence
- auditability records

## What Erebor Can Do

Erebor can show how AI agents acting in cloud, SaaS, browser, or terminal
environments followed customer instructions and policy constraints. This is
especially useful for privileged access, data export/deletion, SaaS interactions,
and incident reconstruction, when those actions are routed through
Erebor-controlled execution paths.

## Artifacts Erebor Could Generate

- cloud PII processor agent activity log
- customer-instruction-to-action trace report
- privileged access review package
- SaaS/subprocessor access evidence report
- PII deletion/export action trace record
- cloud incident timeline for agent activity

## Do Not Claim

Erebor does not replace cloud-provider controls, physical security, contractual
controls, subprocessor disclosure programs, controller obligations, legal
deletion/export determinations, or ISO certification. It covers actions routed
through the governed runtime.
