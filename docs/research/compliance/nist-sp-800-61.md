# NIST SP 800-61

## What It Is For

NIST SP 800-61 Revision 3 is incident response guidance framed as a NIST CSF
2.0 Community Profile. It is a strong fit for Erebor when incident responders
need a reliable timeline of what an agent did.

## Evidence Companies Need

- incident preparation evidence
- detection and analysis records
- containment, eradication, and recovery records
- communications and escalation logs
- lessons learned
- improvement tracking

## What Erebor Can Do

Erebor can provide the incident timeline for AI-agent actions: commands,
browser/SaaS actions, API calls, MCP/tool calls, approvals, denied actions,
data exposure, containment actions, and remediation, when those actions are
routed through Erebor-controlled execution paths. This matters when an agent
causes, worsens, mitigates, or merely touches an incident.

## Artifacts Erebor Could Generate

- AI-agent incident action timeline
- containment command approval log
- browser/SaaS access reconstruction
- denied high-risk action report
- lessons-learned evidence packet
- incident response mapping report

## Do Not Claim

Erebor is not an incident response program, SOC, incident commander, legal
breach decision maker, notification owner, or scope/materiality authority. It
supplies high-quality runtime evidence for agent-related incidents.
