# PCI DSS

## What It Is For

PCI DSS is the payment-card data security standard for entities that store,
process, transmit, or can affect the security of cardholder data environments.
PCI DSS v4.0.1 is the current baseline; v4.0 retired on December 31, 2024, and
many new v4 requirements became effective on March 31, 2025.

## Evidence Companies Need

- cardholder data environment scope
- asset inventory
- access-control and MFA evidence
- admin activity logs
- vulnerability and patch evidence
- change-management approvals
- incident response evidence
- service-provider oversight
- SAQ, ROC, and AOC artifacts where applicable

## What Erebor Can Do

Erebor can govern AI agents operating in or near the cardholder data
environment: browser sessions, terminals, SaaS consoles, tools, approvals,
blocked actions, identity attribution, API/MCP actions, and incident timelines,
when those actions are routed through Erebor-controlled execution paths.

## Artifacts Erebor Could Generate

- AI-agent CDE access review
- governed session audit log
- high-risk command/action approval ledger
- payment workflow change-control evidence pack
- incident timeline for agent activity
- service-provider agent activity report

## Do Not Claim

Erebor does not make an environment PCI compliant, determine PCI scope or
merchant/service-provider level, replace segmentation, encryption,
vulnerability scanning, QSA/ASV validation, SAQ, ROC, or AOC.
